Krita 4.2.1

Apenas unos días después de lanzar la versión 4.2.0 ponemos a disposición la 4.2.1. En ésta se ha resuelto el problema donde después de usar Krita por un tiempo, el dibujar se hace bastante lento, esto por que las entradas de “deshacer” se hacían infinitas. También vienen otros ajustes como el guardar archivos .kra con capas de vectores, donde las imágenes usan letras que no son del “latin1”. Tenemos ademas nuevos contribuyentes que han ayudado a solucionar algunos de estos errores: Carl Olsson y Dmitry Utkin, ¡Gracias!

Lista de problemas y errores de código resueltos:

  • Lentitud después de uso prolongado (BUG:408255, BUG:408133)
  • Cargar capas de vectores cuando la imagen tiene nombre que no es latin1 (BUG:408280)
  • Cuadriculado incorrecto después de cambiar el tamaño del lienzo (BUG:408166)
  • Ventana de propiedades de capas que permanece encima de otras ventanas (BUG:408170)
  • Uso de filtro de posterizado con sRGB
  • Permitir que la animación comience del fotograma 0 (BUG:408198)
  • Seleccionado del opaco en grupos de capas (BUG:408236)
  • Revisión de la fuente de almacén (drive) antes de producir la animación (BUG:408246)
  • Prevención del borrado de capas que estan bloqueadas
  • Copiado y duplicado de las capas (BUG:408028)
  • Transparencia de la ventana de Krita con Qt en Linux (BUG:408225)
  • Retraso del pincel al principio del trazo (BUG:408133)
  • Doblaje/Desdoblaje de la imagen previa de las capas
  • Hacer y deshacer en la guías de pintura (corregido por Tusooa Zhu)
  • Mostrado de guías con Python API (BUG:408113)
  • Consistencia con el tamaño mínimo de los pinceles
  • Consistencia con el valor del espaciado del pincel (BUG:359453)
  • Comportamiento del seleccionado en panel de las capas (BUG:408002)
  • Rotación y amplificación del lienzo en macOS (BUG:404321)
  • Trazos entre objetos de vectores (BUG:407683)
  • Controlador de la herramienta de documentos (BUG:408146)
  • Selección del nombre de la capa al editarse (BUG:400357)

Krita 4.2.1 ahora usa Qt 5.12.3 lo cual debe resolver los problemas con el arrastrar y relocar documentos, pero puede haber problemas con ls escala de la pantalla en Windows ya que el código experimental que resuelve los problemas con Qt trae nuevos conflictos.

Descargas

Windows

En Windows, si se encuentran el cerrado del programa inesperadamente, favor de seguir las siguientes instrucciones (en ingles) para encontrar los errores y así poder entender por qué suceden.

  • 64 bits Windows: krita-x64-4.2.1-setup.exe

  • Portable 64 bits Windows: krita-x64-4.2.1.zip

  • Debug symbols. (Unpack in the Krita installation folder)

  • 32 bits Windows: krita-x86-4.2.1-setup.exe

  • Portable 32 bits Windows: krita-x86-4.2.1.zip

  • Debug symbols. (Unpack in the Krita installation folder)

Linux

  • 64 bits Linux: krita-4.2.1-x86_64.appimage
  • 64 bits Linux G’Mic-Qt plugin appimage.

En ocasiones Firefox trata el enlace como texto, para descargar el paquete simplemente has un clic a la derecha en el mismo enlace.

OSX

  • OSX disk image: krita-4.2.1.dmg

Nota: El controlador de gmic-qt no está disponible en Mac OS X.

Código fuente

  • krita-4.2.1.tar.gz
  • krita-4.2.1.tar.xz
